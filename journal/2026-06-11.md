### Agile V Integration — Decided

WindowConfigurator will be used as the **live experiment ground for Agile V adoption**. The framework will be integrated **one component at a time**, with each component justified before the next is added.

**First component: requirements with IDs**

Before any implementation work begins on Slice D, a formal requirements document with numbered IDs will be written. Each requirement must be testable and scoped. The existing TDD discipline then becomes traceable — failing tests map to requirement IDs.

Nothing else changes in the first pass: same TDD loop, same ADRs, same journal, same PR gate. The only addition is a requirements doc written before the branch is touched.

**Why one at a time:** Every component carries a token cost. Adopting a component means measuring what it produces against what it costs. This is also article content — empirical evidence for the token frugality methodology.

**What comes after (not yet scheduled):**
- Traceability matrix linking requirement IDs to tests and code
- Red Team Verifier pass (independent agent challenge before merge)
- Compliance/audit documentation

Each of these must earn its place before the next is added.
