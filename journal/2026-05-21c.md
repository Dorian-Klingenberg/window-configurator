## Slice 2 — API Key Rotation/Revocation Lifecycle

### What Was Built

**`TenantEntity`** — added two nullable lifecycle fields:
- `ApiKeyExpiresAtUtc` — key rejected after this timestamp
- `ApiKeyRevokedAt` — key permanently rejected

**`ApiKeyAuthorizeFilter`** — updated to check revocation then expiry after finding the tenant by key. Rejection order: missing → invalid → revoked → expired → grant.

**`TenantApiKeyController`** — new controller at `api/v1/tenants`:
- `POST /{id}/api-key/rotate` — generates `Guid.NewGuid().ToString("N")`, clears lifecycle, optional expiry
- `DELETE /{id}/api-key` — idempotent revoke via `??=`

**DTOs**: `RotateApiKeyRequest`, `RotateApiKeyResponse`

## Slice 3 — Webhook Operations Hardening

### What Was Built

**`WebhookRetryBackgroundService : BackgroundService`** — polls `IWebhookRetryProcessor` on configurable interval (`Webhooks:RetryIntervalSeconds`, default 300). Uses `IServiceScopeFactory` to create a fresh scope per tick (required because `IWebhookRetryProcessor` is scoped). Registered via `AddHostedService<WebhookRetryBackgroundService>()` in `Program.cs`.

**`IWebhookDeliveryAttemptRepository.GetStatsAsync()`** — new method returning `WebhookDeliveryStats { Delivered, Failed, Total }`. Implemented in `EfWebhookDeliveryAttemptRepository` with three `CountAsync` queries.

**`GET /api/v1/webhook-deliveries/stats`** — added to `WebhookDeliveriesController` (which now also accepts `IWebhookDeliveryAttemptRepository`). Returns `WebhookDeliveryStatsResponse { delivered, failed, total, asOfUtc }`.

### Tests Added (3)

- `ExecuteAsync_OnEachInterval_CallsRetryProcessor` (background service, 0-second interval)
- `GetStats_WithMixedAttempts_ReturnsCorrectCounts`
- `GetStats_WithNoAttempts_ReturnsZeroCounts`
