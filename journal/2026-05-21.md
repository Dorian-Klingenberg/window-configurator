## Submitted Payload Pricing Discrepancy

User found a live mismatch in the browser: estimated price `$447.75`, authoritative server price `$467.72`.

The exact submitted payload was recovered from the local SQLite configuration snapshot and added as a regression fixture:

- `WindowConfigurator.Tests/Pricing/fixtures/submitted-44775-payload.json`

The failing test was added first:

- `Complete_WithSubmittedTwoSectionPayload_MatchesLegacyClientEstimate`

That regression was later renamed to `Complete_WithSubmittedTwoSectionPayload_PricesBrickmouldFromTopLevelSizing` to make the asserted behavior explicit.

Root cause: the legacy JavaScript pricing engine uses three different measurement pairs:

- frame color pricing uses `osmWidth` / `osmHeight`
- section, grille, SDL, and pane pricing use each section's `width` / `height`
- brickmould pricing uses the payload's top-level `width` / `height`

The C# completion path was using `frameWidth` / `frameHeight` for brickmould pricing. In brickmould sizing mode those differ from the top-level `width` / `height`, which produced the `$19.97` discrepancy.

Fix:

- `WindowPricingInput` now carries explicit brickmould pricing dimensions
- `OrderItemController.BuildPricingInput` maps those from top-level `width` / `height`, falling back to `frameWidth` / `frameHeight`
- `PricingService.BrickmouldPrice` uses the explicit brickmould pricing dimensions
- completion validation checks brickmould pricing-grid support against the same dimensions

## Pricing Grid Alignment To Catalog Maximums

Adjusted pricing data handling so supported catalog dimensions no longer fall past the final pricing breakpoint.

Added `PricingGridAligner` (`WindowConfigurator.Data/Pricing/PricingGridAligner.cs`) and wired it at startup in `Program.cs` immediately after loading `priceInfo.json`.

Behavior:

- For each product line, the aligner reads the corresponding `*ItemTemplate.json` in `AppData`.
- Style pricing grids are extended to each style's catalog `maxWidth`/`maxHeight`.
- Grille, SDL, and pane configuration grids are extended to the product line's maximum section dimensions across styles.
- Brickmould grids are extended to frame max dimensions plus maximum brickmould inset on both sides.
- New breakpoint values are projected by continuing the last local slope in the existing pricing table.

Updated tests:

- Added `PricingGridAlignerTests` to verify breakpoint coverage reaches catalog limits per product line.
- Updated `CompletionValidationServiceTests` so catalog-valid near-maximum dimensions no longer fail with pricing-grid overflow errors.
