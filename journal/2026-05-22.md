## Customer-Facing Demo Direction (Agreed)

We aligned on a customer-facing simulation as the immediate priority:

1. Build a separate mock contractor website as the demo host surface.
2. Add a clear CTA path (for example, "Configure Your Windows") that starts quote setup.
3. Launch WindowConfigurator from that host via session bootstrap (iframe or redirect fallback).
4. Show completion/confirmation back in the host flow using authoritative outputs.

We also agreed to include a **mock contractor CRM portal** in the demo strategy so the narrative is realistic but not blocked by external CRM availability.

Key reason:
- real CRM systems remain important for integration testing, but are too operationally fragile for deterministic demos.

Operational stance:
- mock CRM/website for demo reliability
- real CRM for integration validation
- keep WindowConfigurator API/webhook contracts CRM-agnostic in both cases
