## Tooling Hardening: VS Code Debug Skill Added

To prevent repeated branch-by-branch debug breakage, we added a dedicated agent skill:

- `skills/vscode-debug-launch-core.md`

Why:
- we repeatedly hit the same failure modes:
  - stale app path references after slice switches (for example `MockContractorSite` vs `MockContractorCrm`)
  - workspace-root vs repo-root `.vscode` drift
  - invalid `serverReadyAction.uriFormat` values (must include exactly one `%s`)

Decision:
- codify the known happy path into one reusable skill so future agents can apply the same setup consistently and quickly.

Also updated:
- `AGENTS.md` now registers `vscode-debug-launch` in the available skills list with invocation guidance.
